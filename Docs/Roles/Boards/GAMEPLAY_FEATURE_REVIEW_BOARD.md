# Gameplay Feature Review Board

## Purpose and Roles

Use for new or materially changed player-facing mechanics. Mandatory roles: Game Design Reviewer, Technical Director, Gameplay QA Reviewer, QA Lead, and Producer / Scope Controller. Activate Unity Specialist for Unity work and add UI/UX, Audio, Art, Camera/Occlusion, Performance, or Release review when affected.

## Required Review

- approved player goal, rules, states, scope, and acceptance criteria;
- implementation and dependency map;
- automated and manual evidence for happy, invalid, boundary, rapid-input, recovery, reset/respawn, and regression cases;
- gameplay-camera evidence for visible/feel work;
- performance evidence when representative load changes;
- separate design, technical, gameplay QA, QA Lead, and Producer verdicts.

Hard fails include a missing core requirement, stuck or unrecoverable state, contradictory rules, serious exploit, failed required regression, or an unresolved defect that Work can correct in scope. Human review is reserved for feel, priority, balance alternatives, scope, or genuine product decisions.
